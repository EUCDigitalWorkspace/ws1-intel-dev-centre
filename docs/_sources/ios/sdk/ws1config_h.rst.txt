WS1Config
=========

This object is used to specify various configuration options for Workspace ONE
Intelligence SDK.

Once this object is setup, you can pass it to ``WS1Intelligence.initialize``.
After Workspace ONE Intelligence SDK is initialized, changes to this object will
have no affect.

Creating a Config
-----------------

.. container:: toggle

    .. container:: header

       \+ defaultConfig

    .. container:: discussion

       Creates a new WS1Config object with the default values for the above
       properties. You can modify the config values and pass this object into
       ``[WS1Intelligence enableWithAppID:config:]``

    .. container:: declaration

       Objective-C

       .. code-block:: objective-c

          + (WS1Config *)defaultConfig

       Swift

       .. code-block:: swift

          class func default() -> WS1Config

    .. container:: retVal

       Returns ``WS1Config`` with default values.

Network Insights Configuration
------------------------------

.. container:: toggle

    .. container:: header

       enableServiceMonitoring *Property*

    .. container:: discussion

       Determines whether automatic collection of network traffic data is enabled in the SDK.

       If this flag is set to NO, then no instrumentation will be installed
       and no data will be collected or sent automatically.

       Default value: ``YES`` (enabled)

    .. container:: declaration

       Objective-C

       .. code-block:: objective-c

          @property (nonatomic, assign) BOOL enableServiceMonitoring;

       Swift

       .. code-block:: swift

          var enableServiceMonitoring: Bool

    .. container:: discussion

.. container:: toggle

    .. container:: header

       urlFilters *Property*

    .. container:: discussion

       An array of WS1Filter objects. These objects are used to fine tune
       the collection of Network Insights data by the Workspace ONE Intelligence
       SDK. For example, URLs that may contain sensitive information can be blocked.
       The filters can also be used to preserve URL query parameters, fragments, 
       and parameters. By default these components are removed from the data reported
       to the WS1Intelligence hub.

    .. container:: declaration

       Objective-C

       .. code-block:: objective-c

          @property (nonatomic, strong) NSArray <WS1Filter *> *urlFilters;

       Swift

       .. code-block:: swift

          var urlFilters: Array<WS1Filter>

Monitoring Web Views
--------------------

.. container:: sdk-section-discussion

   Workspace ONE Intelligence has the capability to monitor network traffic
   generated by web views and JavaScript errors generated by WKWebViews.

   Monitoring of WKWebView traffic is disabled by default because use of the 
   WKWebView class has the side effect of calling the class initialize method, 
   which creates a new thread to manage webviews.

   Since Workspace ONE Intelligence cannot prevent these side effects from
   happening and many apps do not use webviews, Network Insights for
   webviews must be explicitly enabled.

.. container:: toggle

    .. container:: header

       monitorWKWebView *Property*

    .. container:: discussion

       Determine whether Network Insights should capture network performance
       information for network calls made through ``WKWebView``.
       Currently only page loads and page transitions are captured.
       Calls made via javascript are currently not captured.

       Default value: ``NO``

    .. container:: declaration

       Objective-C

       .. code-block:: objective-c

          @property (nonatomic, assign) BOOL monitorWKWebView;

       Swift

       .. code-block:: swift

          var monitorWKWebView: Bool

.. _ios_ws1_config_mach:

Toggle MACH Exception Capture
-----------------------------

.. container:: toggle

    .. container:: header

       enableMachExceptionHandling *Property*

    .. container:: discussion

       Enables or disables MACH exception capturing by the SDK.
       This allows capturing additional crashes such as stack overflows.
       Since installing a mach exception handler can interfere with debuggers, 
       this setting will not take effect when a debugger is attached.
       You may choose to disable mach exception handling if you have code that
       already handles mach exceptions.

       Default value: ``YES``

    .. container:: declaration

       Objective-C

       .. code-block:: objective-c

          @property(nonatomic) BOOL enableMachExceptionHandling;

       Swift

       .. code-block:: swift

          var enableMachExceptionHandling: Bool
